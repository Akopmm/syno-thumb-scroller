# syno-thumb-scroller

Auto-scroll the Synology Photos Android app via ADB to force lazy thumbnail loading across the entire photo timeline.

## Problem

The Synology Photos Android app uses lazy loading — thumbnails are only fetched from the NAS when you physically scroll to that part of the timeline. On large libraries (10k+ photos), the app feels slow and unloaded unless you manually scroll through everything.

The thumbnails already exist on the NAS (generated by DSM). The Android client just never requests them until you scroll there.

## Solution

This script uses **ADB (Android Debug Bridge)** to simulate slow upward swipes on the Synology Photos app, triggering the app's own lazy loader to fetch and cache thumbnails — exactly as if a human were scrolling slowly through the timeline.

No reverse-engineering, no API calls, no root required.

## Requirements

- Python 3.10+
- ADB installed (auto-detected from common install locations, or in PATH)
- Android phone with USB debugging enabled and Synology Photos installed

## Setup

### 1. Install ADB

| Platform | Command |
|----------|---------|
| macOS | `brew install android-platform-tools` |
| Linux | `sudo apt install adb` |
| Windows | [Download Platform Tools](https://developer.android.com/tools/releases/platform-tools) |

### 2. Connect your phone

**USB:** Plug in, accept the "Allow USB debugging" prompt on phone.

**WiFi:**
```bash
adb tcpip 5555
adb connect <phone-ip>:5555
```

### 3. Run

```bash
python scroller.py
```

The script will auto-detect ADB, find your device, launch Synology Photos, and start scrolling.

## Usage

```
python scroller.py [options]

Options:
  --device DEVICE       ADB device serial or IP:port (default: auto-detect)
  --swipes N            Total number of swipes (default: 200)
  --pause SECONDS       Pause after each swipe in seconds (default: 3.0)
  --swipe-duration MS   Duration of each swipe in milliseconds (default: 2000)
  --distance PIXELS     Pixels per swipe (default: 800)
  --no-launch           Skip auto-launching the app
  --debug-screenshots   Save a screenshot every 10 swipes
```

## Examples

```bash
# Default — 200 swipes, 3s pause (~17 min)
python scroller.py

# Large library — more swipes, slower pace
python scroller.py --swipes 500 --pause 4

# Specific device over WiFi
python scroller.py --device 192.168.1.42:5555

# Fast mode (thumbnails may not fully load)
python scroller.py --pause 1.5 --swipe-duration 1000

# Debug — save screenshots to verify scrolling
python scroller.py --debug-screenshots --swipes 30
```

## Tuning Guide

| Library size | Recommended swipes | Recommended pause |
|---|---|---|
| < 1,000 photos | 50–100 | 2.5s |
| 1,000–5,000 | 200–300 | 3.0s |
| 5,000–20,000 | 500–800 | 3.5s |
| 20,000+ | 1000+ | 4.0s |

- **Thumbnails not loading?** Increase `--pause` — the NAS needs time to respond.
- **Scrolling too fast/slow?** Adjust `--swipe-duration` (higher = slower, more natural swipe).

## Tips

Keep the screen on during the run:
```bash
adb shell settings put system screen_off_timeout 2147483647
```

Restore after:
```bash
adb shell settings put system screen_off_timeout 60000
```

## Troubleshooting

| Issue | Fix |
|---|---|
| `adb: command not found` | Install ADB — the script also checks common install paths automatically |
| No devices found | Accept the USB debugging prompt on phone, or re-run `adb connect` |
| Swipes do nothing | Make sure the Synology Photos timeline is open and visible |
| Thumbnails still blank | Increase `--pause`, your NAS may be slow |
| Wrong app opens | The package name is `com.synology.projectkailash` — edit `SYNOLOGY_PHOTOS_PACKAGE` in `scroller.py` if yours differs |
